The four rules: memory is scoped to (workspace, clearance) so it never crosses tenants or leaks up the ladder · DLP'd on write (secrets and raw dumps refused) · tamper-evident (hash-chained) · and inert — data, never instruction, so it can't authorize anything. Within-session scratch stays in the ephemeral sandbox and is wiped on teardown; this is the persistent store outside it.
